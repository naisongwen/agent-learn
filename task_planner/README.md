# Claude Task Planner - 任务规划演示

## 项目概述

这是 Claude 核心概念体系中"计划与推理"能力的演示实现。展示了 AI 代理如何进行任务分解、依赖管理、优先级排序和执行规划。

## 核心概念

### 1. 任务分解能力
- 将复杂项目拆分为可管理的子任务
- 定义清晰的任务边界和交付物
- 评估每个任务的工作量和复杂度

### 2. 依赖关系管理
- 识别任务间的前置依赖关系
- 确保执行顺序的正确性
- 处理阻塞和等待状态

### 3. 优先级规划
- 基于业务价值设定任务优先级
- 平衡紧急性和重要性
- 动态调整优先级序列

### 4. 执行路径优化
- 智能推荐最优执行顺序
- 识别并行执行机会
- 管理资源约束和瓶颈

## 功能特性

### 📋 任务管理
- **任务创建**: 完整的任务属性定义
- **状态跟踪**: 从待开始到完成的全流程管理
- **依赖解析**: 自动检测和验证依赖关系
- **优先级排序**: 基于业务规则的智能排序

### 🎯 计划优化
- **时间估算**: 基于历史数据的工时预测
- **路径规划**: 最优执行序列计算
- **资源分配**: 任务到执行者的智能匹配
- **风险识别**: 提前发现潜在阻塞点

### 📊 可视化展示
- **项目仪表板**: 实时项目状态概览
- **甘特图预览**: 时间线和依赖关系可视化
- **进度追踪**: 完成率和效率指标
- **瓶颈分析**: 识别项目关键路径

## 使用方法

### 1. 直接运行演示
```bash
cd agent-learn/task_planner
python demo.py
```

### 2. 作为模块使用
```python
from task_planner.demo import TaskPlanner, Task, TaskStatus

# 创建规划器
planner = TaskPlanner()

# 添加任务
task = Task(
    id="task_001",
    title="环境搭建",
    description="配置开发环境",
    priority=5,
    estimated_hours=2.0
)
planner.add_task(task)

# 获取可执行任务
ready_tasks = planner.get_ready_tasks()
```

项目: 构建一个完整的Web应用程序
任务分解:

├── 测试验证 (最低优先级, 依赖集成)

```
推荐算法:
1. 优先选择依赖已满足的任务

```

## 核心组件

### Task 类
```python
@dataclass
class Task:
    id: str              # 任务唯一标识
    title: str           # 任务标题
    description: str     # 详细描述
    status: TaskStatus   # 任务状态
    priority: int        # 优先级 (1-5)
    estimated_hours: float  # 预估工时
    dependencies: List[str] # 依赖任务ID列表
```

### TaskPlanner 类
```python
class TaskPlanner:
    def add_task(self, task: Task) -> None
    def get_ready_tasks(self) -> List[Task]  # 获取可开始的任务
    def plan_project_timeline(self) -> Dict   # 项目时间线规划
    def visualize_plan(self) -> str          # 可视化展示
```

## 最佳实践

### 1. 任务设计原则
- **MECE原则**: 相互独立，完全穷尽
- **SMART标准**: 具体、可衡量、可达成、相关、有时限
- **原子性**: 每个任务应该是最小可执行单元

### 2. 依赖管理技巧
- 明确定义依赖类型（硬依赖vs软依赖）
- 设置合理的等待超时机制
- 建立依赖解除预案

### 3. 优先级设定方法
- **MoSCoW法则**: Must/Must not, Should, Could, Won't
- **Eisenhower矩阵**: 紧急-重要四象限
- **Kano模型**: 基本型/期望型/兴奋型需求

### 4. 执行优化策略
- 识别关键路径任务
- 平衡串行和并行执行
- 建立缓冲时间和应急方案
